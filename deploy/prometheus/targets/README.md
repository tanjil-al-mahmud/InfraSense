# Prometheus Service Discovery Target Files

This directory contains JSON target files for Prometheus file-based service discovery. These files define the endpoints that Prometheus should scrape for OS-level agent metrics.

## Overview

Prometheus is configured to dynamically discover agent endpoints by reading JSON files from this directory. The files are refreshed every 5 minutes, allowing for dynamic addition/removal of monitored agents without restarting Prometheus.

## Target File Format

Each JSON file follows the Prometheus file-based service discovery format:

```json
[
  {
    "targets": ["<ip>:<port>", "<ip>:<port>"],
    "labels": {
      "device_id": "<unique-device-id>",
      "hostname": "<hostname>",
      "device_type": "<agent-type>",
      "location": "<location>"
    }
  }
]
```

## Target Files

### node_exporter_targets.json
- **Purpose**: Linux servers running node_exporter
- **Default Port**: 9100
- **Scrape Interval**: 15 seconds
- **Metrics**: CPU, RAM, disk, network, system uptime

### windows_exporter_targets.json
- **Purpose**: Windows servers running windows_exporter
- **Default Port**: 9182
- **Scrape Interval**: 15 seconds
- **Metrics**: CPU, RAM, disk, network, system uptime

### lm_sensors_exporter_targets.json
- **Purpose**: Linux machines with lm-sensors hardware monitoring
- **Default Port**: 9255
- **Scrape Interval**: 15 seconds
- **Metrics**: CPU temperature, fan speed, voltage readings

### smartctl_exporter_targets.json
- **Purpose**: Linux machines with smartctl disk SMART monitoring
- **Default Port**: 9633
- **Scrape Interval**: 60 seconds
- **Metrics**: Reallocated sectors, pending sectors, uncorrectable errors, disk temperature, power-on hours

## Dynamic Updates

In production, these JSON files should be automatically generated by a service that:
1. Queries the PostgreSQL Asset_Database for registered devices with agent types
2. Filters devices by agent type (linux_agent, windows_agent)
3. Generates the appropriate target file with device endpoints
4. Writes the JSON file to this directory
5. Prometheus automatically detects changes within 5 minutes

## Manual Updates

For development/testing, you can manually edit these files:

1. Add a new target entry with the agent's IP address and port
2. Include required labels: device_id, hostname, device_type, location
3. Save the file
4. Prometheus will detect the change within 5 minutes (refresh_interval)

## Validation

To validate the JSON format:
```bash
cat node_exporter_targets.json | jq .
```

To check if Prometheus has loaded the targets:
```bash
curl http://localhost:9090/api/v1/targets | jq '.data.activeTargets[] | select(.job=="node-exporter")'
```

## Requirements Validated

This configuration validates the following requirements:
- **Requirement 4.1**: OS-Level Metrics Collection from Linux Servers
- **Requirement 5.1**: OS-Level Metrics Collection from Windows Servers
- **Requirement 32.5**: Hardware Sensor Monitoring via lm-sensors
- **Requirement 33.7**: Disk SMART Monitoring via smartctl
